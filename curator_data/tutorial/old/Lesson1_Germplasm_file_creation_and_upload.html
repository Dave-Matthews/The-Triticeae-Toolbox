This tutorial will guide a curator/user in nomenclature rules, data template aquisition, line name validation, and data upload. Any feedback is welcome!

Section One. Notes and Rules for Germplasm Line Names in The Triticeae Toolbox (T3)

  • Line names must be entered into T3 before any phenotype or genotype data about those lines can be loaded.
  • If PI or CItr/CIHo values exist, please include them in the alias column of the template.
  • Multiple aliases can be used, but currently aliases cannot be duplicated within T3.
  • Separate multiple aliases with commas.
  • Aliases do not have to follow the nomenclature rules of primary line names, i.e. they can have spaces, mixed case, etc.
  • Genotype and phenotype data can be loaded onto T3 using a germplasm line's primary name or alias.

    Macros to convert line names to T3 format are written into an Excel spreadsheet that can be downloaded from the Data Submission page or here.


    Table 1
    Line Names Rules Examples
    1. All upper case Winoka → WINOKA
    2. No spaces. Spaces between two letters or between two numbers can be replaced with an underscore. Remove all spaces between numbers and letters. Red Fife → RED_FIFE
    UC1599 08010/7 → UC1599_08010/7
    Tara 2002 → TARA2002
    3. No single quotes, apostrophes, special characters or double quotes. Château d'Yquem → CHATEAU_DYQUEM
    4. All line names must be at least 3 characters long. If the name is all numbers, it must have at least 4 digits. Substitute the PI number in these cases. K2 → PI486829
    5. Two different lines cannot have the same name. That is, all line names must be unique. Adding a pipe and the PI number will accomplish this. ARAMIR|PI399482
    ARAMIR|PI467786

    Section Three. Validate the need to upload new germplasm lines

    Before uploading line names, determine if they are already on T3.

    Click 'Lines' from the top menu bar and 'Select Lines by Properties' from the dropdown menu.

    Enter line names in the 'Name' text box. A list can also be 'copy and pasted' from an Excel column, text file, etc.

    If the lines records exist in T3, a report will be generated on the lower section on the page.
    If germplasm lines are not on T3, like the example lines MT113001 - MT113003, error messages are generated.
    Line information will therefore be required for the MT lines before genotype or phenotype data can be entered.


      Figure 4. Results of a Line Name search in T3

    Section Four. Fill in the template.

    One of the required items needed on the header of the Line Submission Template is the Breeding Program Code.


      Figure 5. Header of the Line Submission Form Excel Template

    If more than one breeding program's lines are used, separate uploads may be necessary if the lines are not already in T3.

    Two-letter codes are used for barley, three-letter codes are used for wheat breeding programs.

    The easiest way to determine the 2 or 3 letter Breeding Program Code is to click the 'Search by Breeding Programs' dropdown menu on the T3 homepage.

    The code follows the hyphen after the full name of the program, so for example, wheat lines from CIMMYT have the breeding program code CMT, lines from UC Davis will be encoded UCD, etc..


      Figure 6. Dropdown menu from the T3 homepage to view T-CAP breeding programs and codes

    Section Five. Upload the Data

    Log in as a Curator/User.
    If you do not have Curator privileges, please contact Victoria Carollo Blake for more information.

    Once a curator is successfully logged onto T3, additional tabs will appear on the top menu bar.

    Click the Curation tab and select 'Add/Edit Lines'.


      Figure 7. T3 Homepage with Curator tabs

    Name the completed template to indicate the germplasm set and save as an Excel .xls file.
    The notes do not need to be removed from the worksheet.

    In Internet Explorer or Firefox, click the Browse button to select the file to upload.
    In Google Chrome or Safari, click 'Choose File' to select the file to upload.
    Click 'Upload Line File' button (not browser-specific) to load the selected file into T3.


      Figure 8. New Line Upload page for T3 Curators

    On a successful upload, two boxes will appear (Fig. 9).

    The upper box reports how T3 will read the file. In this example, five germplasm entries were read from the input file. Three new lines, e.g. TESTUPLOADLINE1, are added with minimal input data. Two existing lines, CALEDONIA and CAYUGA have new aliases in this file.

    The lower box reports the lines that will be added to T3 in the left column (i.e. TESTUPLOADLINE1 - 3) and the lines that T3 will edit (CAYUGA and CALEDONIA) in the right column.

    IMPORTANT: If there are any lines in the right column, the data in your file will overwrite the existing data about those lines. If that is not what you intended to do, click 'Cancel'.

    Click 'Accept' to load the data into T3.


      Figure 9. Line Upload Report

    If there are problems with the data, errors produce messages that should clearly indicate the editing needed.
    Note that if an error occurs, nothing has loaded.
    Occasionally an error can occur after the 'Accept' button is clicked.
    Please contact the development team if this happens.
